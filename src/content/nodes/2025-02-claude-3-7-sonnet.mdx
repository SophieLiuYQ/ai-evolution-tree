## Hybrid vs separate-model approach

| Approach | OpenAI | Anthropic |
|---|---|---|
| Standard model | GPT-4o | Claude 3.7 Sonnet (default) |
| Reasoning model | o1, o1-pro, o3 | Claude 3.7 Sonnet (extended-thinking mode) |
| Switch granularity | per-API-endpoint | per-request flag |
| Reasoning trace | hidden (summary only) | fully visible |
| Pricing | reasoning ~5-10× more | reasoning ~3-5× more (token-based) |

The hybrid approach has UX advantages (one model abstraction for developers) but architectural costs (the same weights have to handle both modes well). The empirical evidence as of mid-2025 is that hybrid can compete on quality but with somewhat higher inference cost in standard mode.

## SWE-bench Verified — the headline benchmark

SWE-bench Verified is a curated subset of real GitHub issues from popular Python repos; the model must produce a code patch that passes the project's test suite.

| Model | Date | SWE-bench Verified |
|---|---|---|
| GPT-4 (Mar 2023) | 2023-03 | 1.7% |
| Claude 3 Opus | 2024-03 | 11.7% |
| GPT-4o | 2024-05 | 33.2% (with agent scaffolding) |
| Claude 3.5 Sonnet | 2024-06 | 49.0% |
| Claude 3.5 Sonnet (Oct update) | 2024-10 | 53.1% |
| o3-mini | 2025-01 | 49.3% |
| **Claude 3.7 Sonnet (extended)** | **2025-02** | **70.3%** |
| Claude 4 Sonnet | 2025-05 | 73.6% |
| Claude 4 Opus | 2025-05 | 79.4% |

The SWE-bench trajectory is the cleanest single-benchmark story of LLM progress 2023-25 — a 41× improvement in two years on a task that maps to real economic value.

## Claude Code release

Anthropic shipped Claude Code (a CLI agentic coding tool, eponymous with this codebase's likely tooling reader) the same week as Claude 3.7 Sonnet. It used the model in extended-thinking mode by default and demonstrated multi-step plan→edit→test loops that would consume 50,000+ tokens per task. The economic message: at $0.003/1k input, a Claude Code session might cost $5-50 depending on task scope — vs the engineering hour it replaces.
