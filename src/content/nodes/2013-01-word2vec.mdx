## The famous analogy result

```
vec("king") − vec("man") + vec("woman") ≈ vec("queen")
vec("Paris") − vec("France") + vec("Italy") ≈ vec("Rome")
vec("walking") − vec("walked") + vec("swam") ≈ vec("swimming")
```

Geometric operations on the embedding space correspond to semantic operations on words. This wasn't designed in — it emerged from the training objective. The result was widely interpreted as evidence that learned representations capture meaningful structure beyond what was explicitly supervised.

## What word2vec started

| Year | Successor | What changed |
|---|---|---|
| 2014 | GloVe | matrix-factorization variant, comparable quality |
| 2016 | fastText | sub-word tokenization, multilingual |
| 2018 | ELMo / BERT | contextual embeddings (word vectors depend on sentence) |
| 2021 | CLIP | cross-modal embeddings (text + image in same space) |
| 2022 | OpenAI Embeddings API | embeddings as a paid product category |
| 2023+ | RAG | embeddings + vector search as the dominant LLM augmentation pattern |

Modern LLM tokenizers (BPE, SentencePiece) and contextual embeddings have technically replaced word2vec for production — but the conceptual frame ("represent everything as vectors in a learned space") is the durable contribution.
