## Why MCP succeeded where prior standards stalled

Earlier attempts at standardizing AI tool use (LangChain Hub, OpenAI's function-calling extensions, AI Plugins) didn't achieve cross-vendor adoption. MCP did, for several reasons:

1. **Anthropic positioned it as a community standard from day one** — open spec, permissive license, neutral name (no "Anthropic" branding in the spec)
2. **Reference implementations were strong** — Python and TypeScript SDKs at launch worked out of the box
3. **Claude Desktop integration** drove initial mindshare among developers
4. **OpenAI adopted it in March 2025**, which sealed it as cross-vendor de facto standard
5. **Google adopted it later in 2025**, completing the major-lab triangle

## What an MCP server looks like

```typescript
// Conceptual MCP server (TypeScript)
import { McpServer } from "@modelcontextprotocol/sdk";

const server = new McpServer({ name: "linear" });

server.tool("create-issue", {
  description: "Create a Linear issue",
  parameters: { title: "string", body: "string", projectId: "string" },
  handler: async ({ title, body, projectId }) => {
    return await linearClient.issueCreate({ title, body, projectId });
  },
});

server.resource("issue", {
  description: "Read a Linear issue by ID",
  read: async (uri) => {
    const issueId = uri.split("/").pop();
    return await linearClient.issue(issueId);
  },
});

server.serve();
```

Once running, this server's capabilities are available to any MCP-compatible AI app (Claude Desktop, Cursor, ChatGPT, etc.) the user authorizes to connect.

## Lineage

MCP is the explicit protocolization of the ReAct (2022-10) pattern. ReAct established that "model + tool calls + observation loop" was the right abstraction; MCP standardizes the wire format and discovery protocol so that any model can use any tool. The protocol layer sits above any specific model and below any specific application.
