## What it is

A 1.5B-parameter decoder-only transformer trained on WebText (curated outbound links from Reddit). Same architecture as GPT-1, ~13× scaled. The contribution is empirical: showed that scaling improved zero-shot transfer to multiple downstream tasks without any task-specific fine-tuning.

## Versus GPT-1 (8 months earlier)

| Dimension | GPT-1 (2018-06) | GPT-2 (2019-02) |
|---|---|---|
| Params | 117M | 1.5B (×13) |
| Training data | BookCorpus ~1B tok | WebText ~10B tok |
| Context | 512 | 1024 |
| LAMBADA | required fine-tune | 63.2% zero-shot |
| Approach | pretrain + fine-tune | pretrain + zero-shot |
| Release | weights public | staged release controversy |

The leap from "fine-tune for every task" to "zero-shot via prompting" was the actual scientific bet that paid off in GPT-3.

## The staged release controversy

OpenAI released only the smallest 117M variant initially, citing concerns about misuse. They released 345M, then 762M over the following months, and finally the full 1.5B in November 2019 — with a research paper documenting that the predicted harms (mass disinformation campaigns, automated abuse) had not materialized at the smaller-model release tiers.

This established a template — "responsible disclosure for AI capabilities" — that every subsequent frontier lab (Anthropic, DeepMind, Meta) has adopted in some form. It also established the criticism — "safetywashing" — that the model's deployment cadence was driven as much by marketing as risk.
