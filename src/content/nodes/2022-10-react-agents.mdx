## What it is

A prompting framework where the model emits an interleaved sequence of:

```
Thought: I need to find when X happened.
Action: search("X event date")
Observation: X happened in 1987.
Thought: Now I should figure out who was president then.
Action: search("US president 1987")
Observation: Ronald Reagan.
Thought: ...
Action: finish("Ronald Reagan")
```

The model decides at each step whether to think more or take an action; an external runtime executes the action and feeds the observation back. This is the "agent loop" — reasoning, acting, observing, repeat.

## What ReAct combined that wasn't combined before

| Capability | Pre-ReAct | ReAct |
|---|---|---|
| LM reasoning | CoT (Wei 2022) | ✓ inherited |
| LM tool use | restricted to fine-tuned single-tool pipelines | ✓ general |
| Reasoning + tool use in one trajectory | ✗ | ✓ |
| Trace-based debugging | ✗ | ✓ (each thought visible) |

The contribution wasn't any single technical innovation — it was the integration into a single coherent trajectory that the model itself orchestrated.

## What ReAct birthed

| Successor | Date | Inheritance |
|---|---|---|
| LangChain agents | 2023-Q1 | productized the ReAct loop as a Python framework |
| OpenAI function calling | 2023-06 | API-native tool schema (formalizes the action format) |
| AutoGPT / BabyAGI | 2023-04 | autonomous multi-step ReAct with goal decomposition |
| ToolFormer (Meta) | 2023-02 | trains the model to insert API calls during pretraining |
| Anthropic tool use | 2024-04 | first-class tool-use API with structured streaming |
| MCP (Model Context Protocol) | 2024-11 | open protocol for tool / context server integration |
| Devin / Cursor / Claude Code | 2024-25 | productized agentic IDE / dev workflows |

Every modern agentic system inherits the core ReAct loop, even when not explicitly cited.
