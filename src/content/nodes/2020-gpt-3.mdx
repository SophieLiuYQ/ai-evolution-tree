## What it is

GPT-3 is a 175-billion-parameter autoregressive transformer language model trained on ~300B tokens of internet text. Architecturally it is mostly identical to GPT-2 (a decoder-only transformer with sparse attention variants); the contribution is **scale** plus the empirical demonstration of **in-context learning**.

In-context learning: rather than fine-tuning the model on a new task, you describe the task in the prompt and provide a few examples ("few-shot"). The model then completes the pattern. This worked surprisingly well across translation, question-answering, arithmetic, code generation, and dozens of other tasks the model was never explicitly trained on.

## How it works

The training objective is unchanged from GPT-1/GPT-2 — next-token prediction:

$$
\mathcal{L} = -\sum_{t} \log p_\theta(x_t \mid x_{<t})
$$

The breakthrough is that scaling this objective monotonically improves performance on a broad range of downstream tasks, *and* unlocks qualitatively new behaviors (in-context learning, basic arithmetic, code synthesis) at threshold sizes.

Few-shot prompting in practice:

```
Translate English to French:

sea otter => loutre de mer
plush giraffe => girafe en peluche
cheese =>
```

GPT-3 completes with `fromage`. No fine-tuning was performed for translation; the capability was elicited by the prompt.

## How to use it

GPT-3 itself is no longer a current model (OpenAI deprecated the original `davinci` line). The interface — and the techniques — survive in every modern LLM API:

```python
import openai
client = openai.OpenAI()

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "You are a Python tutor."},
        {"role": "user", "content": "Explain decorators with one example."},
    ],
)
print(response.choices[0].message.content)
```

The `chat.completions` interface is post-GPT-3 (introduced with ChatGPT/GPT-3.5), but the underlying mechanics — token-by-token autoregressive sampling from a transformer — are unchanged.

## SWOT

- **Strengths** — General-purpose: a single model handles dozens of tasks via prompting. Few-shot reduces or eliminates per-task data collection. API-as-product economics — the model is the SaaS.
- **Weaknesses** — Hallucinates confidently. Limited reliability for tasks requiring exact recall or multi-step reasoning. Expensive: ~$5M to train (2020 dollars), millions per month to serve at scale.
- **Opportunities** — Tool use (function calling), retrieval augmentation (RAG), and inference-time reasoning (chain-of-thought, o1-style) all extend GPT-3-class models without retraining.
- **Threats** — Open-weight competitors (Llama 3, Mistral) erode pricing power. Specialized small models (1B–10B) outperform generalist 175B models on narrow tasks.

## Limitations (still mostly unsolved as of 2026)

- **Hallucination** — fluent text that asserts false facts.
- **No persistent memory** across sessions without external scaffolding.
- **Limited multi-step reasoning** — partially addressed by chain-of-thought (next era) and inference-time compute (o1 / o3).
- **Compute cost** — inference at scale dominates the unit economics of every LLM product.
- **Training data provenance** — copyright and licensing remain legally unresolved.

## Why GPT-3 created an industry

Three things happened together:

1. **Capability surprise** — the model was much better at much more than its predecessors. The few-shot demos went viral.
2. **API distribution** — OpenAI made the model available as a paid API rather than open-sourcing it. This created a business.
3. **Scaling laws made capital allocation legible** — Kaplan et al. (2020) gave investors a quantitative framework: $X compute + $Y data + $Z parameters → $W loss. Once you can forecast capability from spend, you can underwrite billion-dollar training runs.

The combination — a viable product, a viable business model, and an investable scaling thesis — is what GPT-3 contributed beyond any single technical idea.
